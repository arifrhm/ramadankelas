# Contributing ke RamadanKelas

Terima kasih atas ketertarikan Anda untuk berkontribusi ke RamadanKelas!

## 🚀 Cara Berkontribusi

1. **Fork Repository**
   ```bash
   gh repo fork arifrhm/ramadankelas
   ```

2. **Clone Fork Anda**
   ```bash
   git clone https://github.com/<username>/ramadankelas.git
   cd ramadankelas
   ```

3. **Buat Branch Baru**
   ```bash
   git checkout -b fitur/nama-fitur
   ```

4. **Lakukan Perubahan**
   - Edit file yang diperlukan
   - Pastikan code mengikuti style guide
   - Add tests jika perlu

5. **Commit Perubahan**
   ```bash
   git add .
   git commit -m "Tambah: deskripsi fitur"
   ```

6. **Push ke Fork**
   ```bash
   git push origin fitur/nama-fitur
   ```

7. **Buat Pull Request**
   - Buka repository asli
   - Klik "New Pull Request"
   - Isi template PR dengan deskripsi perubahan

## 📝 Style Guide

### TypeScript
- Gunakan TypeScript untuk semua file baru
- Hindari penggunaan `any` type
- Gunakan type inference saat memungkinkan

### Naming Conventions
- **Components**: PascalCase (misal: `CourseCard.tsx`)
- **Functions**: camelCase (misal: `fetchCourses`)
- **Constants**: UPPER_SNAKE_CASE (misal: `API_BASE_URL`)

### Code Formatting
- Gunakan 2 spasi untuk indentasi
- Gunakan single quotes untuk strings
- Tambahkan trailing comma di objects/arrays

## 🐛 Reporting Bugs

Sebelum report bug, pastikan:
- [ ] Saya sudah search issue yang ada
- [ ] Saya sudah reproduce bug
- [ ] Saya sudah cek environment requirements

Format bug report:

```markdown
## Deskripsi
Deskripsikan bug secara jelas

## Steps untuk Reproduce
1. Buka halaman ini
2. Klik tombol ini
3. Lakukan aksi ini

## Expected Behavior
Apa yang seharusnya terjadi?

## Actual Behavior
Apa yang terjadi?

## Environment
- OS: [misal: Ubuntu 20.04]
- Node version: [misal: v18.17.0]
- Browser: [misal: Chrome 120]
```

## 💡 Fitur Request

Jika punya ide fitur baru:

1. Cek issue yang ada untuk menghindari duplikat
2. Gunakan template issue "Feature Request"
3. Jelaskan dengan detail mengapa fitur ini penting
